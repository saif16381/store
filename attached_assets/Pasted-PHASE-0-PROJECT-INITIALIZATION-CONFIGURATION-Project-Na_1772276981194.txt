PHASE 0 — PROJECT INITIALIZATION & CONFIGURATION
Project Name: Pakstore
Description: A multi-vendor e-commerce marketplace (similar to Etsy) where users can register, create their own stores, list products, and buyers can browse, search, add to cart, and place orders. The entire platform is a serverless-first, thick-client Single Page Application.

Initialize the project with the following exact tech stack:
- React 18 with TypeScript (strict mode enabled)
- Vite as the build tool (latest stable version)
- Tailwind CSS for utility-first styling
- shadcn/ui component library built on Radix UI primitives
- Framer Motion for all animations and page transitions
- wouter for client-side routing (do NOT use react-router)
- Zustand for global client-side UI state (cart state, auth UI state, sidebar toggles, modals)
- TanStack Query v5 for all server/async state management, caching, and Firestore data synchronization
- Zod for all form validation and data schema validation
- Firebase SDK (v9+ modular) for Authentication and Cloud Firestore database
- Cloudinary for all image and media uploads and optimized delivery

Project structure should follow a feature-based architecture:
src/
├── app/                    (App entry, providers, global layout)
│   ├── App.tsx
│   ├── Providers.tsx       (QueryClientProvider, AuthProvider wrappers)
│   └── Layout.tsx
├── features/
│   ├── auth/               (login, register, password reset, auth hooks, auth store)
│   ├── stores/             (store creation, store dashboard, store profile page)
│   ├── products/           (product listing, product detail, product CRUD)
│   ├── cart/               (cart drawer/page, cart store with Zustand)
│   ├── orders/             (order placement, order history, order tracking)
│   ├── search/             (search bar, search results, filters)
│   └── home/               (landing page, featured products, categories)
├── shared/
│   ├── components/         (reusable UI: Navbar, Footer, ProductCard, StoreCard, Rating, etc.)
│   ├── hooks/              (useAuth, useFirestore, useCloudinary, useDebounce, etc.)
│   ├── lib/                (firebase.ts config, cloudinary.ts config, utils.ts)
│   ├── schemas/            (Zod schemas: userSchema, productSchema, storeSchema, orderSchema)
│   ├── types/              (TypeScript interfaces and types)
│   └── constants/          (routes, categories, config values)
├── stores/                 (Zustand store files: cartStore.ts, uiStore.ts)
└── api/                    (Serverless functions directory for SEO, sitemaps, OG tags)

Configure Vite with:

PHASE 1 — FIREBASE SETUP, AUTHENTICATION & USER DATA MODEL
Set up Firebase configuration in src/shared/lib/firebase.ts using the modular v9+ SDK. Initialize Firebase App, Auth, and Firestore exports.

Implement the full authentication system under src/features/auth/:

1. AUTH PROVIDER (context):
   - Create an AuthProvider that wraps the app and listens to Firebase onAuthStateChanged
   - Expose current user, loading state, and auth methods via a custom useAuth hook
   - On auth state change, fetch or create the corresponding User document in Firestore

2. USER DATA MODEL (Firestore collection: "users"):
   - uid (string, matches Firebase Auth UID)
   - email (string)
   - displayName (string)
   - photoURL (string, optional)
   - role ("buyer" | "seller" | "admin")
   - storeId (string | null — references their store document if they have one)
   - createdAt (Firestore Timestamp)
   - updatedAt (Firestore Timestamp)
   
   Define this as a Zod schema in src/shared/schemas/userSchema.ts and a TypeScript type in src/shared/types/

3. AUTH PAGES & FORMS:
   - /login page: Email/password login + Google OAuth sign-in button. Validate with Zod. Show loading and error states. Animate with Framer Motion fade-in.
   - /register page: Name, email, password, confirm password. On success, create Firestore user doc with role "buyer". Redirect to home.
   - /forgot-password page: Email input, sends Firebase password reset email.
   - All forms use shadcn/ui Input, Button, Label, Card components.
   - All forms have proper TypeScript typing, Zod validation with real-time error messages.

4. PROTECTED ROUTES:
   - Create a ProtectedRoute wrapper component that redirects unauthenticated users to /login
   - Create a SellerRoute wrapper that additionally checks if user role is "seller" and has a storeId

5. ZUSTAND AUTH UI STORE:
   - Manage UI-only auth state: isLoginModalOpen, isAuthLoading, authError
   - This is separate from the actual Firebase auth state

PHASE 8 — FIRESTORE SECURITY RULES & FINAL CONFIGURATION
Provide Firestore Security Rules (as a firestore.rules file or documentation) with these policies:

1. USERS collection:
   - Any authenticated user can read any user profile (public data)
   - Users can only update their own document
   - Users can create their own document (on registration)

2. STORES collection:
   - Anyone (even unauthenticated) can read active stores
   - Only authenticated users can create a store (and only if they don't already have one)
   - Only the store owner can update their store
   - Stores cannot be deleted (only deactivated)

3. PRODUCTS collection:
   - Anyone can read active products
   - Only the store owner can create products in their store (storeId must match)
   - Only the store owner can update/delete their products
   - Validate required fields in rules

4. ORDERS collection:
   - Only authenticated users can create orders (buyerId must match auth UID)
   - Buyers can read their own orders (buyerId == auth UID)
   - Sellers can read orders containing their storeId in items array
   - Only sellers can update order status (limited field update)
   - Orders cannot be deleted

5. REVIEWS subcollection:
   - Anyone can read reviews
   - Only authenticated users can create (one per user per product)
   - Users can only update/delete their own reviews

Also provide Firestore indexes that will be needed:
- products: (category ASC, createdAt DESC) for category browsing
- products: (storeId ASC, createdAt DESC) for store product listing
- products: (isActive ASC, category ASC, price ASC) for filtered search
- orders: (buyerId ASC, createdAt DESC) for buyer order history

Create a Cloudinary configuration note in the README explaining:
- Create an unsigned upload preset named "pakstore_uploads"

- Set max file size: 5MB
- Enable auto-format and auto-quality transformations
- Use Cloudinary URL transformations for responsive images (w_400 for thumbnails, w_800 for product detail)
